## 1. 当前架构现状分析

### 1.1 解码器：实为切空间欧式评分

**关键代码**（`hyperbolic_decoder.py` `HyperbolicConvTransE.forward()`）：

```python
# 将实体嵌入映射到切空间
entity_tangent = HyperbolicOps.log_map_zero(entity_embedding, self.c)
# IMPROVED: Use leaky tanh (scale + shift) for better gradient flow
entity_tangent = 0.9 * torch.tanh(entity_tangent) + 0.1 * entity_tangent

# 所有后续操作均在切空间（欧式空间）进行
e1_embedded = entity_tangent[triplets[:, 0]].unsqueeze(1)
rel_embedded = rel_embedding[triplets[:, 1]].unsqueeze(1)
...
scores = torch.mm(x, entity_tangent.transpose(1, 0))
```

**问题根源**：

- 解码时首先调用 `log_map_zero` 将所有实体嵌入投影到切空间，随后的卷积、全连接、矩阵乘积都在欧式空间中进行。
- 这等价于：在欧式空间跑 ConvTransE，只是输入做了一次 `arctanh` 变换。
- 双曲空间的核心优势——**指数级层级容量**与**测地距离关系**——在解码阶段完全被丢弃。
- 关系嵌入（`rel_embedding`）始终在欧式切空间，与实体双曲结构存在空间不一致。

### 1.2 GNN 编码器：切空间聚合 + 投影

**关键代码**（`hyperbolic_layers.py` `HyperbolicUnionRGCNLayer.forward()`）：

```python
# Step 1: Map to tangent space（切空间操作起点）
h_tangent = HyperbolicOps.log_map_zero(h_hyper, self.c)
g.ndata['h_tangent'] = h_tangent

# 消息传递：全部在切空间（欧式）进行
g.update_all(lambda x: self.msg_func(x), fn.sum(msg='msg', out='h_tangent'), ...)

# Step 3: Map back to hyperbolic space（只在最后映射）
h_hyper_new = HyperbolicOps.exp_map_zero(h_new, self.c)
```

**问题根源**：

- 消息传递（message passing）完全在欧式切空间执行：`msg = F.linear(node + relation, W)`，求和聚合也是欧式加法。
- 这与原始 Euclidean RGCN 的计算图等价，只是多了首尾的 `log_0/exp_0` 变换。
- 半径感知权重 `radius_weight = exp(-|r_i - r_j|)` 有一定改进，但并未从本质上修改聚合操作的几何性质。
- 双曲空间的层级传播特性（指数增长的邻域容量）在聚合阶段未被利用。

### 1.3 核心差距总结

| 模块 | 当前实现 | 理想实现 |
|------|---------|---------|
| 解码器评分 | 切空间内积（欧式） | 双曲距离 / Möbius 变换 |
| 关系变换 | 欧式向量相加 | Möbius 加法 + 旋转 |
| 消息传递 | 欧式线性变换 + 求和 | Möbius 矩阵乘法 + Fréchet 均值 |
| 聚合操作 | 欧式求和 | Einstein 中点 / 加权 Fréchet 均值 |
| 注意力权重 | 欧式内积 | 双曲距离函数 |

## 2. 方向一：真双曲解码器设计方案

### 2.1 MuRP 风格双曲距离解码器

**参考来源**：Balazevic et al., "Multi-Relational Poincaré Graph Embeddings" (NeurIPS 2019)

#### 2.1.1 数学原理

对于三元组 $(s, r, o)$，评分函数完全在 Poincaré 球内进行：

$$f(s, r, o) = -d_\mathbb{H}^2\!\left(R_r \otimes_c \mathbf{h}_s \oplus_c \mathbf{t}_r,\ \mathbf{h}_o\right) + b_s + b_o$$

其中：
- $\otimes_c$：Möbius 矩阵-向量乘法（对角矩阵近似），$M \otimes_c x = \exp_0(M \cdot \log_0(x))$
- $\oplus_c$：Möbius 加法（关系平移）
- $d_\mathbb{H}$：双曲距离，$d_\mathbb{H}(x,y) = \frac{2}{\sqrt{c}}\operatorname{arctanh}\!\left(\sqrt{c}\,\|\!-\!x \oplus_c y\|\right)$
- $R_r$：关系旋转（对角矩阵，仅实部，维度 $d$）
- $\mathbf{t}_r$：关系平移（Poincaré 球上的点）
- $b_s, b_o$：实体偏置

#### 2.1.2 实现细节

**新增参数**（替换 `HyperbolicConvTransE` 中的 `conv1`, `fc` 等）：

```python
# 关系旋转矩阵（对角形式，实际为向量）
self.rel_diag = nn.Parameter(torch.Tensor(num_relations, embedding_dim))
nn.init.uniform_(self.rel_diag, -1.0, 1.0)

# 关系平移（在切空间初始化，前向中映射至双曲空间）
self.rel_trans = nn.Parameter(torch.Tensor(num_relations, embedding_dim))
nn.init.uniform_(self.rel_trans, -1e-3, 1e-3)

# 实体偏置
self.entity_bias = nn.Parameter(torch.zeros(num_entities))
```

**前向计算**：

```python
def forward(self, entity_embedding, rel_embedding, triplets, mode="train"):
    # 1. 取主体嵌入（已在 Poincaré 球上）
    s_emb = entity_embedding[triplets[:, 0]]            # (B, d)

    # 2. 关系旋转：对角 Möbius 矩阵乘法
    rot = self.rel_diag[triplets[:, 1]]                 # (B, d)
    # Möbius matvec: exp_0(rot * log_0(s))
    s_tangent = HyperbolicOps.log_map_zero(s_emb, self.c)
    rot_s_tangent = rot * s_tangent                     # 对角乘法
    rot_s = HyperbolicOps.exp_map_zero(rot_s_tangent, self.c)

    # 3. 关系平移：Möbius 加法
    t_r = HyperbolicOps.exp_map_zero(
        self.rel_trans[triplets[:, 1]], self.c          # 平移映射到球面
    )
    query = HyperbolicOps.mobius_add(rot_s, t_r, self.c)  # (B, d)

    # 4. 计算与所有候选实体的双曲距离
    # 扩展 query: (B, 1, d)，候选: (1, N, d)
    query_expand = query.unsqueeze(1)                   # (B, 1, d)
    cand_expand = entity_embedding.unsqueeze(0)         # (1, N, d)

    # 向量化 Möbius 距离计算
    neg_query = -query_expand.expand(len(triplets), self.num_entities, -1)
    cand = cand_expand.expand(len(triplets), self.num_entities, -1)
    diff = HyperbolicOps.mobius_add(
        neg_query.reshape(-1, self.embedding_dim),
        cand.reshape(-1, self.embedding_dim),
        self.c
    ).reshape(len(triplets), self.num_entities, -1)
    dist_sq = torch.sum(diff ** 2, dim=-1)              # (B, N)

    # 5. 加偏置，转换为相似度分数
    scores = -dist_sq + self.entity_bias[triplets[:, 0]].unsqueeze(1) \
             + self.entity_bias.unsqueeze(0)
    return scores
```

> **注意**：向量化距离计算内存开销为 $O(B \times N \times d)$，在大规模实体集（$N > 10^4$）时需分块计算。

#### 2.1.3 优缺点

| 优点 | 缺点 |
|------|------|
| 评分完全在双曲空间，保留几何结构 | 距离计算内存开销大 |
| 数学形式简洁，可与现有 Poincaré 操作无缝衔接 | 旋转仅为对角矩阵，表达能力有限 |
| 实现复杂度低 | 大规模实体集需分块计算 |

### 2.2 RotH 风格旋转双曲解码器

**参考来源**：Chami et al., "Low-Dimensional Hyperbolic Knowledge Graph Embeddings" (ACL 2020)

#### 2.2.1 数学原理

RotH 在 Poincaré 球上使用 Givens 旋转，将关系建模为旋转 + 平移：

$$f(s, r, o) = -d_\mathbb{H}^2\!\left(\text{Rot}_r\!\left(\mathbf{h}_s\right) \oplus_c \mathbf{t}_r,\ \mathbf{h}_o\right) + b_s + b_o$$

其中 $\text{Rot}_r$ 是一系列 2D Givens 旋转（每对相邻维度一个旋转角）：

$$\text{Rot}_r(x) = \exp_0\!\left(\mathbf{G}_r \cdot \log_0(x)\right)$$

$$\mathbf{G}_r = \begin{pmatrix} \cos\theta_1 & -\sin\theta_1 & & \\ \sin\theta_1 & \cos\theta_1 & & \\ & & \ddots & \\ & & & \cos\theta_{d/2} & -\sin\theta_{d/2} \\ & & & \sin\theta_{d/2} & \cos\theta_{d/2} \end{pmatrix}$$

#### 2.2.2 实现细节

```python
class HyperbolicRotH(nn.Module):
    """
    RotH 风格旋转双曲解码器。
    在 Poincaré 球上使用 Givens 旋转 + Möbius 平移 + 双曲距离评分。
    """

    def __init__(self, num_entities, num_relations, embedding_dim, c=0.01, dropout=0.0):
        super().__init__()
        self.num_entities = num_entities
        self.embedding_dim = embedding_dim
        self.c = c
        assert embedding_dim % 2 == 0, "embedding_dim 必须为偶数（Givens 旋转要求）"
        self.half_dim = embedding_dim // 2

        # Givens 旋转角（每个关系 d/2 个角度）
        self.rel_rot = nn.Parameter(torch.Tensor(num_relations, self.half_dim))
        nn.init.uniform_(self.rel_rot, -math.pi, math.pi)

        # 关系平移（切空间初始化）
        self.rel_trans = nn.Parameter(torch.Tensor(num_relations, embedding_dim))
        nn.init.uniform_(self.rel_trans, -1e-3, 1e-3)

        # 实体偏置
        self.entity_bias = nn.Parameter(torch.zeros(num_entities))
        self.dropout = nn.Dropout(dropout)

    def givens_rotation(self, x, angles):
        """
        对 x 应用 Givens 旋转。
        Args:
            x: (B, d) 切空间向量
            angles: (B, d/2) 旋转角
        Returns:
            旋转后的向量 (B, d)
        """
        # 分成偶数/奇数维度对
        x1 = x[:, 0::2]   # (B, d/2)
        x2 = x[:, 1::2]   # (B, d/2)
        cos = torch.cos(angles)
        sin = torch.sin(angles)
        out1 = cos * x1 - sin * x2
        out2 = sin * x1 + cos * x2
        # 交织合并
        out = torch.stack([out1, out2], dim=2).reshape(x.shape)
        return out

    def forward(self, entity_embedding, rel_embedding, triplets, mode="train"):
        s_emb = entity_embedding[triplets[:, 0]]        # (B, d)，Poincaré 球
        r_angles = self.rel_rot[triplets[:, 1]]         # (B, d/2)
        t_r_tangent = self.rel_trans[triplets[:, 1]]    # (B, d)，切空间

        # 1. 旋转：在切空间应用 Givens 旋转，再映射回双曲空间
        s_tangent = HyperbolicOps.log_map_zero(s_emb, self.c)
        rot_s_tangent = self.givens_rotation(s_tangent, r_angles)
        rot_s = HyperbolicOps.exp_map_zero(rot_s_tangent, self.c)

        # 2. 平移：Möbius 加法
        t_r = HyperbolicOps.exp_map_zero(t_r_tangent, self.c)
        query = HyperbolicOps.mobius_add(rot_s, t_r, self.c)  # (B, d)

        # 3. 双曲距离评分（分块计算，避免 OOM）
        scores = self._batch_hyperbolic_distance_scores(
            query, entity_embedding, triplets, mode
        )
        return scores

    def _batch_hyperbolic_distance_scores(self, query, entity_embedding, triplets, mode):
        """
        分块计算双曲距离评分，避免 OOM。
        """
        B = query.shape[0]
        N = self.num_entities

        # 分块大小，可根据 GPU 内存调整
        chunk_size = min(512, N)
        all_scores = []

        for start in range(0, N, chunk_size):
            end = min(start + chunk_size, N)
            cand = entity_embedding[start:end]           # (C, d)
            # 广播计算 Möbius 距离
            q_exp = query.unsqueeze(1).expand(-1, end - start, -1).reshape(-1, self.embedding_dim)
            c_exp = cand.unsqueeze(0).expand(B, -1, -1).reshape(-1, self.embedding_dim)
            neg_q = -q_exp
            diff = HyperbolicOps.mobius_add(neg_q, c_exp, self.c)
            dist_sq = torch.sum(diff ** 2, dim=-1).reshape(B, -1)   # (B, C)
            chunk_scores = -dist_sq + self.entity_bias[start:end].unsqueeze(0)
            all_scores.append(chunk_scores)

        scores = torch.cat(all_scores, dim=1)    # (B, N)
        # 加主体偏置
        scores = scores + self.entity_bias[triplets[:, 0]].unsqueeze(1)
        return scores
```

#### 2.2.3 优缺点

| 优点 | 缺点 |
|------|------|
| Givens 旋转保持等距性，不破坏双曲度量 | 参数量比 MuRP 略多（多了旋转角参数） |
| 能建模反对称关系（旋转方向可反转） | 旋转仅在同对维度内，不跨维度交互 |
| 分块计算支持大规模实体集 | 实现略复杂 |

### 2.3 AttH 风格注意力双曲解码器

**参考来源**：Chami et al., "Low-Dimensional Hyperbolic Knowledge Graph Embeddings" (ACL 2020) § AttH

#### 2.3.1 数学原理

AttH 通过**注意力机制**在旋转与反射之间自适应插值，关系变换为：

$$\mathbf{h}_r(s) = a_r \cdot \text{Rot}_r(\mathbf{h}_s) + (1 - a_r) \cdot \text{Ref}_r(\mathbf{h}_s)$$

$$a_r = \sigma\!\left(w_r^\top \cdot \text{concat}(\log_0(\mathbf{h}_s), \mathbf{r})\right) \in [0,1]$$

评分函数仍为：

$$f(s, r, o) = -d_\mathbb{H}^2\!\left(\mathbf{h}_r(s) \oplus_c \mathbf{t}_r,\ \mathbf{h}_o\right) + b_s + b_o$$

其中 $\text{Ref}_r$ 为 Givens 反射（利用反射角参数）。

#### 2.3.2 实现细节

```python
class HyperbolicAttH(nn.Module):
    """
    AttH 风格注意力双曲解码器。
    通过注意力机制在旋转与反射间自适应插值，在 Poincaré 球上进行距离评分。
    """

    def __init__(self, num_entities, num_relations, embedding_dim, c=0.01, dropout=0.0):
        super().__init__()
        self.num_entities = num_entities
        self.embedding_dim = embedding_dim
        self.c = c
        assert embedding_dim % 2 == 0
        self.half_dim = embedding_dim // 2

        # Givens 旋转角
        self.rel_rot = nn.Parameter(torch.Tensor(num_relations, self.half_dim))
        nn.init.uniform_(self.rel_rot, -math.pi, math.pi)

        # Givens 反射角
        self.rel_ref = nn.Parameter(torch.Tensor(num_relations, self.half_dim))
        nn.init.uniform_(self.rel_ref, -math.pi, math.pi)

        # 注意力权重向量（2d -> 1 的线性变换）
        self.attn_weight = nn.Parameter(torch.Tensor(num_relations, 2 * embedding_dim))
        nn.init.xavier_uniform_(self.attn_weight)

        # 关系平移
        self.rel_trans = nn.Parameter(torch.Tensor(num_relations, embedding_dim))
        nn.init.uniform_(self.rel_trans, -1e-3, 1e-3)

        # 实体偏置
        self.entity_bias = nn.Parameter(torch.zeros(num_entities))
        self.dropout = nn.Dropout(dropout)

    def givens_rotation(self, x, angles):
        x1 = x[:, 0::2]; x2 = x[:, 1::2]
        cos = torch.cos(angles); sin = torch.sin(angles)
        return torch.stack([cos * x1 - sin * x2, sin * x1 + cos * x2], 2).reshape(x.shape)

    def givens_reflection(self, x, angles):
        x1 = x[:, 0::2]; x2 = x[:, 1::2]
        cos = torch.cos(angles); sin = torch.sin(angles)
        return torch.stack([cos * x1 + sin * x2, sin * x1 - cos * x2], 2).reshape(x.shape)

    def forward(self, entity_embedding, rel_embedding, triplets, mode="train"):
        s_emb = entity_embedding[triplets[:, 0]]   # (B, d)
        r_rot = self.rel_rot[triplets[:, 1]]        # (B, d/2)
        r_ref = self.rel_ref[triplets[:, 1]]        # (B, d/2)
        t_r_tan = self.rel_trans[triplets[:, 1]]    # (B, d)

        # 1. 切空间投影
        s_tangent = HyperbolicOps.log_map_zero(s_emb, self.c)   # (B, d)

        # 2. 旋转与反射
        rot_s = self.givens_rotation(s_tangent, r_rot)
        ref_s = self.givens_reflection(s_tangent, r_ref)

        # 3. 注意力权重：基于主体嵌入 + 关系嵌入
        attn_input = torch.cat([s_tangent, rel_embedding[triplets[:, 1]]], dim=-1)  # (B, 2d)
        a_r = torch.sigmoid(
            torch.sum(self.attn_weight[triplets[:, 1]] * attn_input, dim=-1, keepdim=True)
        )   # (B, 1)

        # 4. 插值混合
        mixed_tangent = a_r * rot_s + (1 - a_r) * ref_s   # (B, d)
        mixed_hyp = HyperbolicOps.exp_map_zero(mixed_tangent, self.c)

        # 5. 关系平移
        t_r = HyperbolicOps.exp_map_zero(t_r_tan, self.c)
        query = HyperbolicOps.mobius_add(mixed_hyp, t_r, self.c)   # (B, d)

        # 6. 双曲距离评分
        scores = self._batch_hyperbolic_distance_scores(query, entity_embedding, triplets)
        return scores

    def _batch_hyperbolic_distance_scores(self, query, entity_embedding, triplets,
                                          chunk_size=512):
        B = query.shape[0]; N = self.num_entities
        all_scores = []
        for start in range(0, N, chunk_size):
            end = min(start + chunk_size, N)
            cand = entity_embedding[start:end]
            q_e = query.unsqueeze(1).expand(-1, end - start, -1).reshape(-1, self.embedding_dim)
            c_e = cand.unsqueeze(0).expand(B, -1, -1).reshape(-1, self.embedding_dim)
            diff = HyperbolicOps.mobius_add(-q_e, c_e, self.c)
            dist_sq = torch.sum(diff ** 2, dim=-1).reshape(B, -1)
            all_scores.append(-dist_sq + self.entity_bias[start:end].unsqueeze(0))
        scores = torch.cat(all_scores, dim=1)
        return scores + self.entity_bias[triplets[:, 0]].unsqueeze(1)
```

### 2.4 方案对比与选型建议

| 方案 | 数学完整性 | 实现复杂度 | 内存开销 | 适用场景 |
|------|-----------|-----------|---------|---------|
| **MuRP** | ★★★☆☆（对角旋转） | ★★☆☆☆（低） | 中（需分块） | 快速原型验证、小规模数据 |
| **RotH** | ★★★★☆（Givens 旋转） | ★★★☆☆（中） | 中 | 反对称关系丰富的数据集 |
| **AttH** | ★★★★★（旋转+反射+注意力） | ★★★★☆（较高） | 中偏高 | 复杂关系结构、精度要求高 |

**建议优先级**：
1. **首选 RotH**：数学完整，实现可控，Givens 旋转在 TKGC 上已验证有效，适合作为双曲解码器的第一个替代方案。
2. **次选 AttH**：精度更高但实现复杂，适合在 RotH 基础上进一步提升。
3. **MuRP** 作为消融对照基线。

**与现有代码的集成**：在 `hyperbolic_decoder.py` 中新增以上类，并在 `hyperbolic_model.py` 的 `__init__` 中通过 `decoder_name` 参数分支加载（如 `"roth"`, `"atth"`, `"murp"`）。关系解码器（`rdecoder`）也应对应替换为 `HyperbolicRotH` 的关系预测变体。

## 3. 方向二：新型 GNN 编码器方案

### 3.1 全双曲图卷积网络（FHNN-GCN）

**参考来源**：Zhang et al., "Fully Hyperbolic Neural Networks" (ACL 2022)

#### 3.1.1 数学原理

FHNN 将所有矩阵乘法替换为 Möbius 矩阵-向量乘法，将求和聚合替换为 Einstein 中点（双曲空间的 Fréchet 均值近似）：

**消息计算**（Möbius 矩阵乘法）：

$$\mathbf{m}_{u \to v}^r = W_r \otimes_c \mathbf{h}_u = \exp_0\!\left(W_r \cdot \log_0(\mathbf{h}_u)\right)$$

**聚合**（Einstein 中点，双曲空间加权均值）：

$$\text{Agg}_v = \bigoplus_c^{\text{Einstein}}\!\left\{ \alpha_{uv}^r \cdot \mathbf{m}_{u \to v}^r \right\}_{u \in \mathcal{N}(v)}$$

Einstein 中点公式：

$$\bigoplus_c^E \{w_i, x_i\} = \frac{\sum_i w_i \lambda_c(x_i) x_i}{\sum_i w_i \lambda_c(x_i)}, \quad \lambda_c(x) = \frac{2}{1 - c\|x\|^2}$$

（其中 $\lambda_c(x)$ 为 Lorentz 因子）

**更新**（Möbius 加法 + 非线性）：

$$\mathbf{h}_v' = \sigma_\otimes\!\left(\text{Agg}_v \oplus_c \text{Loop}_v\right)$$

其中 $\sigma_\otimes(x) = \exp_0(\sigma(\log_0(x)))$ 为双曲空间的非线性激活。

#### 3.1.2 实现细节

```python
class FHNNLayer(nn.Module):
    """
    全双曲图卷积层（FHNN）。
    所有操作均在 Poincaré 球上直接进行，使用 Möbius 操作和 Einstein 中点聚合。
    """

    def __init__(self, in_feat, out_feat, num_rels, c=0.01,
                 activation=None, self_loop=False, dropout=0.0):
        super().__init__()
        self.c = c
        self.activation = activation
        self.self_loop = self_loop

        # 每个关系一个变换矩阵（可用基分解节省参数）
        self.rel_weight = nn.Parameter(torch.Tensor(num_rels, in_feat, out_feat))
        nn.init.xavier_uniform_(self.rel_weight.reshape(num_rels, -1).unsqueeze(0))

        if self_loop:
            self.loop_weight = nn.Parameter(torch.Tensor(in_feat, out_feat))
            nn.init.xavier_uniform_(self.loop_weight)

        self.dropout = nn.Dropout(dropout) if dropout > 0 else None
        self.rel_emb = None   # 由外部在 forward 前注入

    @staticmethod
    def einstein_midpoint(embeddings, weights, c, eps=1e-6):
        """
        在 Poincaré 球上计算加权 Einstein 中点。
        Args:
            embeddings: (N, d)，Poincaré 球上的点
            weights: (N,)，权重（已归一化）
            c: 曲率
        Returns:
            中点坐标 (d,)，在 Poincaré 球上
        """
        norms_sq = torch.sum(embeddings ** 2, dim=-1)    # (N,)
        lambda_c = 2.0 / (1.0 - c * norms_sq + eps)     # Lorentz 因子 (N,)
        numerator = torch.sum(
            (weights * lambda_c).unsqueeze(-1) * embeddings, dim=0
        )
        denominator = torch.sum(weights * lambda_c) + eps
        midpoint = numerator / denominator
        return HyperbolicOps.project_to_ball(midpoint, c)

    def msg_func(self, edges):
        """Möbius 矩阵-向量乘法消息。"""
        W = self.rel_weight[edges.data['type']]    # (E, in, out)
        h_src = edges.src['h_hyper']               # (E, in)，Poincaré 球
        # Möbius matvec：exp_0(W @ log_0(h))
        h_tangent = HyperbolicOps.log_map_zero(h_src, self.c)   # (E, in)
        msg_tangent = torch.bmm(h_tangent.unsqueeze(1), W).squeeze(1)  # (E, out)
        # 加入关系嵌入（切空间相加后映射回双曲）
        if self.rel_emb is not None:
            rel = self.rel_emb[edges.data['type']]
            # 仅取前 out_feat 维或投影
            msg_tangent = msg_tangent + rel[:, :msg_tangent.shape[-1]]
        msg_hyper = HyperbolicOps.exp_map_zero(msg_tangent, self.c)
        return {'msg_hyper': msg_hyper, 'norm': edges.dst['norm']}

    def reduce_func(self, nodes):
        """Einstein 中点聚合。"""
        msgs = nodes.mailbox['msg_hyper']        # (N, K, d)，K 为邻居数
        norms = nodes.mailbox['norm']            # (N, K, 1)
        B, K, d = msgs.shape
        # 权重为归一化因子
        weights = norms.squeeze(-1) / (norms.squeeze(-1).sum(dim=1, keepdim=True) + 1e-6)

        # 逐节点计算 Einstein 中点
        results = []
        for i in range(B):
            results.append(
                self.einstein_midpoint(msgs[i], weights[i], self.c)
            )
        return {'h_agg': torch.stack(results)}    # (N, d)

    def forward(self, g, h_hyper, rel_emb, prev_h=None):
        self.rel_emb = rel_emb
        g.ndata['h_hyper'] = h_hyper

        # Einstein 中点聚合
        g.update_all(self.msg_func, self.reduce_func)
        h_agg = g.ndata.pop('h_agg')    # (N, d)，Poincaré 球

        # 自环：Möbius matvec
        if self.self_loop:
            h_tangent = HyperbolicOps.log_map_zero(h_hyper, self.c)
            loop_tangent = torch.mm(h_tangent, self.loop_weight)
            loop_hyp = HyperbolicOps.exp_map_zero(loop_tangent, self.c)
            # Möbius 加法合并聚合结果与自环
            h_new = HyperbolicOps.mobius_add(h_agg, loop_hyp, self.c)
        else:
            h_new = h_agg

        # 双曲非线性激活
        if self.activation is not None:
            h_tan = HyperbolicOps.log_map_zero(h_new, self.c)
            h_tan = self.activation(h_tan)
            h_new = HyperbolicOps.exp_map_zero(h_tan, self.c)

        if self.dropout is not None:
            h_tan = HyperbolicOps.log_map_zero(h_new, self.c)
            h_tan = self.dropout(h_tan)
            h_new = HyperbolicOps.exp_map_zero(h_tan, self.c)

        return h_new
```

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 所有操作均在双曲空间，几何一致性最强 | Einstein 中点逐节点计算，效率较低 |
| 保留层级结构的完整性 | 参数量较大（每关系一个矩阵） |
| 与双曲解码器天然配合 | 数值稳定性要求较高 |

### 3.2 Lorentz 模型图卷积网络（LGCN）

**参考来源**：Zhang et al., "Lorentzian Graph Convolutional Networks" (WWW 2021)

#### 3.2.1 数学原理

Lorentz/Hyperboloid 模型比 Poincaré 球**数值更稳定**，适合深层 GNN。

Lorentz 模型定义：

$$\mathcal{L}^{d,c} = \{x \in \mathbb{R}^{d+1} : \langle x, x \rangle_\mathcal{L} = -1/c,\ x_0 > 0\}$$

其中 Minkowski 内积 $\langle x, y \rangle_\mathcal{L} = -x_0 y_0 + \sum_{i=1}^d x_i y_i$。

Lorentz 模型线性层：

$$h' = \exp_o^{\mathcal{L}}\!\left(W \cdot \log_o^{\mathcal{L}}(h)\right)$$

Lorentz 聚合（Lorentz 加权质心）：

$$\text{Agg} = \exp_o^{\mathcal{L}}\!\left(\sum_i \alpha_i \log_o^{\mathcal{L}}(h_i)\right)$$

#### 3.2.2 与现有代码的衔接

在 `hyperbolic_ops.py` 中新增 Lorentz 模型操作类 `LorentzOps`，包含：

- `to_lorentz(x, c)`: Poincaré 球 → Lorentz 模型转换
- `to_poincare(x, c)`: Lorentz 模型 → Poincaré 球转换
- `lorentz_log_map(x, base, c)`: Lorentz 对数映射
- `lorentz_exp_map(v, base, c)`: Lorentz 指数映射
- `lorentz_centroid(embeddings, weights, c)`: Lorentz 加权质心

```python
class LorentzOps:
    """Lorentz/Hyperboloid 模型的几何操作。"""

    @staticmethod
    def inner_product(x, y, keepdim=False):
        """Minkowski 内积：<x,y>_L = -x0*y0 + sum(xi*yi, i>=1)"""
        # x,y shape: (..., d+1), 第 0 维为时间分量
        return -x[..., :1] * y[..., :1] + torch.sum(x[..., 1:] * y[..., 1:],
                                                      dim=-1, keepdim=keepdim)

    @staticmethod
    def norm(x, c=1.0, keepdim=False):
        """Lorentz 范数（空间分量的欧式范数）"""
        return torch.norm(x[..., 1:], dim=-1, keepdim=keepdim)

    @staticmethod
    def to_lorentz(x, c=0.01):
        """Poincaré 球点 → Lorentz 模型点。"""
        # x: (..., d)，Poincaré 球
        x_norm_sq = torch.sum(x ** 2, dim=-1, keepdim=True)
        denom = 1.0 - c * x_norm_sq
        x0 = (1.0 + c * x_norm_sq) / (denom + 1e-6) / math.sqrt(c)
        xi = 2.0 * x / (denom + 1e-6) / math.sqrt(c)
        return torch.cat([x0, xi], dim=-1)    # (..., d+1)

    @staticmethod
    def to_poincare(y, c=0.01):
        """Lorentz 模型点 → Poincaré 球点。"""
        # y: (..., d+1)，Lorentz 模型
        return y[..., 1:] / (y[..., :1] * math.sqrt(c) + 1.0)

    @staticmethod
    def lorentz_centroid(embeddings, weights, c=0.01, eps=1e-6):
        """
        Lorentz 加权质心（Fréchet 均值的一阶近似）。
        embeddings: (N, d+1)，Lorentz 模型
        weights: (N,)，非负权重
        """
        w = weights / (weights.sum() + eps)
        centroid = torch.sum(w.unsqueeze(-1) * embeddings, dim=0)
        # 投影回 Lorentz 流形
        # 对时间分量保证 x0 > 0 且满足 <x,x>_L = -1/c
        centroid = centroid / torch.sqrt(
            torch.clamp(-LorentzOps.inner_product(centroid, centroid, keepdim=True) * c,
                        min=eps)
        )
        return centroid
```

在 `hyperbolic_layers.py` 中新增 `LorentzRGCNLayer`，使用 Lorentz 模型进行消息传递和聚合，在最终输出时转回 Poincaré 球。这样做的好处是：

1. 内部计算在 Lorentz 模型（数值更稳定）；
2. 外部接口保持 Poincaré 球格式，与其他模块兼容；
3. 仅需在层首尾各转换一次。

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 数值稳定性明显优于 Poincaré 球 | 需要新增 Lorentz 操作代码 |
| 支持更深的层堆叠（梯度更稳定） | Poincaré↔Lorentz 转换有轻微开销 |
| 与现有 Poincaré 球接口可桥接 | 维度增加 1（d → d+1） |

### 3.3 双曲图注意力网络（HGAT）

**参考来源**：Zhang et al., "Hyperbolic Graph Attention Network" (TPAMI 2021)

#### 3.3.1 数学原理

将图注意力机制引入双曲空间，注意力权重基于**双曲距离**计算：

$$e_{ij}^r = \text{LeakyReLU}\!\left(a_r^\top \cdot \log_0\!\left(\mathbf{h}_i^r \oplus_c (-\mathbf{h}_j)\right)\right)$$

$$\alpha_{ij}^r = \frac{\exp(e_{ij}^r)}{\sum_{k \in \mathcal{N}(i)} \exp(e_{ik}^r)}$$

聚合使用 Einstein 中点（或加权 Möbius 加法近似）：

$$\mathbf{h}_i' = \sigma_\otimes\!\left(\bigoplus_c^E \{\alpha_{ij}^r,\ W_r \otimes_c \mathbf{h}_j\}_{j \in \mathcal{N}(i)}\right)$$

与纯欧式 GAT 相比，**关键差异**：
- 注意力计算使用双曲几何（`mobius_add` + `log_0`）
- 聚合使用 Einstein 中点而非欧式加权求和

#### 3.3.2 实现要点

```python
class HGATLayer(nn.Module):
    """
    双曲图注意力层（HGAT）。
    注意力权重基于双曲距离，聚合使用 Einstein 中点。
    """

    def __init__(self, in_feat, out_feat, num_rels, num_heads=4, c=0.01,
                 activation=None, dropout=0.0, concat_heads=True):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = out_feat // num_heads if concat_heads else out_feat
        self.c = c
        self.concat_heads = concat_heads

        # 每个关系、每个注意力头的权重
        self.rel_weight = nn.Parameter(
            torch.Tensor(num_rels, num_heads, in_feat, self.head_dim)
        )
        nn.init.xavier_uniform_(self.rel_weight.reshape(-1, in_feat, self.head_dim))

        # 注意力向量：作用于 log_0(h_i ⊕ (-h_j))
        self.attn_vec = nn.Parameter(torch.Tensor(num_rels, num_heads, self.head_dim))
        nn.init.xavier_uniform_(self.attn_vec)

        self.leaky_relu = nn.LeakyReLU(0.2)
        self.activation = activation
        self.dropout = nn.Dropout(dropout) if dropout > 0 else None
        self.rel_emb = None

    def msg_func(self, edges):
        """计算消息和注意力原始得分。"""
        W = self.rel_weight[edges.data['type']]       # (E, H, in, head_dim)
        h_src = edges.src['h_hyper']                  # (E, in)
        h_dst = edges.dst['h_hyper']                  # (E, in)

        # Möbius 变换：对每个注意力头
        h_src_t = HyperbolicOps.log_map_zero(h_src, self.c)   # (E, in)
        msgs = []
        for head in range(self.num_heads):
            Wh = torch.bmm(h_src_t.unsqueeze(1), W[:, head]).squeeze(1)   # (E, head_dim)
            msgs.append(HyperbolicOps.exp_map_zero(Wh, self.c))

        # 注意力原始得分：基于 h_i ⊕ (-h_j) 的对数映射
        neg_dst = -h_dst
        diff = HyperbolicOps.mobius_add(
            h_src.view(-1, h_src.shape[-1]),
            neg_dst.view(-1, h_dst.shape[-1]),
            self.c
        )
        diff_tangent = HyperbolicOps.log_map_zero(diff, self.c)  # (E, in)

        # 投影到 head_dim 后计算得分（简化：取前 head_dim 维）
        diff_proj = diff_tangent[:, :self.head_dim]
        attn_scores = [
            self.leaky_relu(
                torch.sum(self.attn_vec[edges.data['type'], h] * diff_proj, dim=-1)
            )
            for h in range(self.num_heads)
        ]   # 每个 head: (E,)

        return {
            'msgs': torch.stack(msgs, dim=1),              # (E, H, head_dim)
            'attn_raw': torch.stack(attn_scores, dim=1),   # (E, H)
            'norm': edges.dst['norm']
        }

    def reduce_func(self, nodes):
        """Softmax 注意力 + Einstein 中点聚合。"""
        msgs = nodes.mailbox['msgs']        # (N, K, H, head_dim)
        attn = nodes.mailbox['attn_raw']    # (N, K, H)
        alpha = torch.softmax(attn, dim=1)  # (N, K, H)

        N, K, H, d = msgs.shape
        agg_heads = []
        for h in range(H):
            # Einstein 中点聚合（权重为注意力 + 度归一化因子）
            weights = alpha[:, :, h]   # (N, K)
            head_msgs = msgs[:, :, h, :]  # (N, K, d)
            # 逐节点计算中点
            head_results = []
            for i in range(N):
                head_results.append(
                    FHNNLayer.einstein_midpoint(head_msgs[i], weights[i], self.c)
                )
            agg_heads.append(torch.stack(head_results))   # (N, d)

        if self.concat_heads:
            h_agg = torch.cat(agg_heads, dim=-1)   # (N, H*d)
        else:
            # 平均所有注意力头（在切空间）
            agg_tan = [HyperbolicOps.log_map_zero(h, self.c) for h in agg_heads]
            h_agg = HyperbolicOps.exp_map_zero(
                torch.stack(agg_tan).mean(0), self.c
            )
        return {'h_agg': h_agg}

    def forward(self, g, h_hyper, rel_emb, prev_h=None):
        self.rel_emb = rel_emb
        g.ndata['h_hyper'] = h_hyper
        g.update_all(self.msg_func, self.reduce_func)
        h_new = g.ndata.pop('h_agg')

        if self.activation is not None:
            h_tan = HyperbolicOps.log_map_zero(h_new, self.c)
            h_new = HyperbolicOps.exp_map_zero(self.activation(h_tan), self.c)
        if self.dropout is not None:
            h_tan = HyperbolicOps.log_map_zero(h_new, self.c)
            h_new = HyperbolicOps.exp_map_zero(self.dropout(h_tan), self.c)
        return h_new
```

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 注意力机制能区分重要邻居 | 多头机制内存开销较高 |
| 双曲距离注意力保留几何意义 | Einstein 中点计算效率低（待优化） |
| 对不同重要性邻居有自适应能力 | 超参（注意力头数）需调整 |

### 3.4 时序感知双曲注意力图网络（TH-GAT）

这是针对**时序知识图谱**场景的专项改进，将时间信息融入双曲图注意力：

#### 3.4.1 核心思想

在 HGAT 基础上引入**时序衰减注意力**：距离当前时间越远的历史快照，注意力权重指数衰减。

$$\alpha_{ij}^{r,t} = \text{softmax}\!\left(e_{ij}^r \cdot \exp(-\lambda \cdot \Delta t)\right)$$

其中 $\Delta t$ 为历史时间差，$\lambda$ 为可学习的衰减率。

同时，将**时间编码**（Time Encoding）融入双曲节点嵌入：

$$\mathbf{h}_e^{(t)} = \text{Möbius-Add}\!\left(\mathbf{h}_e,\ \text{TimeEnc}(t)\right)$$

时间编码可使用 `TimeSformer` 风格的可学习傅里叶编码：

$$\text{TimeEnc}(t)_k = \cos\!\left(\omega_k \cdot t + \phi_k\right), \quad k = 1, \dots, d/2$$

#### 3.4.2 实现要点

```python
class TemporalHyperbolicTimeEncoding(nn.Module):
    """双曲空间中的时间编码模块。"""

    def __init__(self, dim, c=0.01, max_time=3650):
        super().__init__()
        self.c = c
        self.dim = dim
        # 可学习频率和相位
        self.omega = nn.Parameter(torch.randn(dim // 2) * 0.1)
        self.phi = nn.Parameter(torch.zeros(dim // 2))
        # 时间衰减率（全局）
        self.decay_rate = nn.Parameter(torch.tensor(0.1))

    def forward(self, timestamps):
        """
        Args:
            timestamps: (T,) 时间戳（归一化到 [0,1]）
        Returns:
            时间编码 (T, d)，在切空间中，可通过 exp_map_zero 映射到双曲空间
        """
        t = timestamps.unsqueeze(-1)   # (T, 1)
        cos_enc = torch.cos(self.omega * t + self.phi)   # (T, d/2)
        sin_enc = torch.sin(self.omega * t + self.phi)   # (T, d/2)
        enc = torch.cat([cos_enc, sin_enc], dim=-1)       # (T, d)
        # 缩小幅度以避免投影越界
        enc = enc * 0.01
        return HyperbolicOps.exp_map_zero(enc, self.c)

    def temporal_attention_weight(self, delta_t):
        """
        时序衰减权重。
        Args:
            delta_t: (B,) 时间差
        Returns:
            衰减权重 (B,)
        """
        return torch.exp(-torch.abs(self.decay_rate) * delta_t)
```

    from hyperbolic_src.hyperbolic_decoder import HyperbolicMuRP

    self.rdecoder = HyperbolicMuRP(num_rels, num_ents, h_dim, c=c, dropout=input_dropout)

# ===== 编码器选择 =====

## 6. 实验验证方案

### 6.1 基准对比

| 模型名称 | 编码器 | 解码器 | 备注 |
|---------|-------|-------|------|
| RE-GCN (baseline) | RGCN（欧式） | ConvTransE（欧式） | 官方结果 |
| Hyp-RGCN-E | Hyperbolic UVRGCN | ConvTransE（切空间） | 当前实现 |
| Hyp-RGCN-MuRP | Hyperbolic UVRGCN | MuRP（双曲） | 阶段一验证 |
| Hyp-RGCN-RotH | Hyperbolic UVRGCN | RotH（双曲） | 阶段一目标 |
| LGCN-RotH | LGCN（Lorentz） | RotH（双曲） | 阶段二目标 |
| HGAT-RotH | HGAT（双曲注意力） | RotH（双曲） | 阶段三目标 |
| TH-GAT-AttH | TH-GAT（时序） | AttH（双曲） | 阶段四目标 |

### 6.2 评估指标

- **MRR**（Mean Reciprocal Rank）：filtered 版本
- **Hits@1/3/10**：精确命中率
- **训练稳定性**：loss 曲线平滑度、NaN 频率
- **数值健康度**：嵌入均值/方差、边界点比例（`pct_near_boundary`）
- **双曲结构保持度**：曲率分布、层级分离度（度数 vs 半径相关性）

### 6.3 消融实验设计

```
实验组 A（解码器消融，编码器固定为当前 Hyperbolic UVRGCN）：
  A1：ConvTransE（切空间，当前） → 对照
  A2：MuRP（双曲距离）
  A3：RotH（旋转 + 双曲距离）
  A4：AttH（注意力 + 双曲距离）

实验组 B（编码器消融，解码器固定为 RotH）：
  B1：Hyperbolic UVRGCN（切空间聚合，当前）→ 对照
  B2：LGCN（Lorentz 模型聚合）
  B3：HGAT（双曲注意力聚合）
  B4：TH-GAT（时序感知双曲注意力）

实验组 C（最优组合验证）：
  C1：最优编码器 + 最优解码器
  C2：RE-GCN baseline（最终对比）
```
